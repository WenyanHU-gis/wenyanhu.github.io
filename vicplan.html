VicPlan Definition Plan

GeoprocessingPythonData Governance

The VicPlan Definition Plan project involved the comprehensive migration and optimization of critical geoprocessing tools from legacy environments to the modern ArcGIS Pro 3.x framework. This work was essential for maintaining Victoria's planning scheme amendment capabilities.

Key Responsibilities:

  • Refactored legacy Python scripts and ArcGIS Pro Add-ins to ensure compatibility with Python 3 and the latest ArcGIS Pro APIs.
  • Designed automated data validation workflows to ensure the integrity of planning scheme spatial data during the translation process.
  • Collaborated with mapping teams to define requirements for custom toolsets that improve the speed and accuracy of definition plan production.
  • Optimized geoprocessing performance, reducing the execution time of complex spatial buffering and overlay operations.
  • Developed a local Python tool in ArcGIS Pro to handle complex geometry issues when zone or overlay geometries were too complex to process in the application, providing a reliable fallback workflow.

Example of the optimized planning scheme tool interface.

Impact:

Successful transition to a modern GIS environment ensured the continuity of the state's planning operations and provided mapping teams with more reliable, higher-performance tools for daily spatial data management tasks.
